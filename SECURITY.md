# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in NekoTerm, please report it responsibly:

1. **Do NOT** open a public issue on GitHub
2. Send an email to the repository maintainer with:
   - A description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

3. Allow reasonable time for the vulnerability to be addressed before disclosing publicly

## Scope

Security concerns may include:
- Credential or token leakage
- Unsafe handling of user input
- Dependency vulnerabilities
- Unsafe subprocess execution

## Out of Scope

The following are **not** considered security vulnerabilities:
- Provider websites changing their HTML structure (this is expected and handled via updates)
- Provider websites becoming unavailable
- Content availability (NekoTerm does not host any content)

## Acknowledgments

We appreciate responsible disclosure and will credit reporters who report vulnerabilities responsibly.
